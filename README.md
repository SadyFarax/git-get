# 🚀 دانلودر چندمنظوره گیتهاب اکشنز (GitHub Actions Mass Downloader)

یک اسکریپت قدرتمند و رسمی که در **گیتهاب اکشنز** اجرا می‌شود و با نوشتن چند خط دستور ساده، می‌توانید فایل‌ها را دانلود مستقیم، مخازن Git (مثل مدل‌های Hugging Face) را با پشتیبانی از LFS کلون کنید و **سایت‌های کامل را آینه‌سازی (Mirror)** نمایید.  
خروجی نهایی همه چیز را در قالب یک فایل **ZIP** دریافت می‌کنید.

این ابزار به‌ویژه برای شرایطی طراحی شده که دسترسی به بسیاری از سایت‌ها محدود یا فیلتر است و شما نیاز دارید محتوای مورد نظر خود را از طریق زیرساخت گیتهاب دریافت کنید.

---

## 📌 پیش‌نیاز: فورک کردن مخزن (Fork)

برای استفاده از این دانلودر، **ابتدا باید این مخزن را Fork کنید** تا یک نسخه از آن در اکانت گیتهاب خودتان داشته باشید.  
سپس تمام تغییرات و اجرای دستورات در Fork شما انجام می‌شود.

---

## 🔧 دستورات قابل قبول

| دستور     | کاربرد                           | فرمت کلی                                                      | مثال |
|-----------|----------------------------------|----------------------------------------------------------------|------|
| `wget`    | دانلود مستقیم فایل               | `wget <لینک> [o=نام‌خروجی]`                                   | `wget https://example.com/file.zip o=data.zip` |
| `git`     | کلون مخزن Git (مدل‌های HF)       | `git <آدرس> [b=شاخه] [d=پوشه] [lfs=true] [depth=1]`           | `git https://huggingface.co/bert-base-uncased b=main d=bert lfs=true` |
| `mirror`  | آینه‌سازی (کپی کامل) یک سایت      | `mirror <آدرس> [o=پوشه] [depth=1000]`                         | `mirror https://docs.python.org/3/ o=py_docs depth=500` |

**توضیح پارامترهای اختیاری:**
- برای `wget`:  
  `o=...` : مسیر و نام خروجی (نسبت به پوشهٔ خروجی).  
- برای `git`:  
  `b=...` : نام شاخه (پیش‌فرض: شاخهٔ اصلی مخزن).  
  `d=...` : پوشه‌ای که مخزن درون آن کلون شود.  
  `lfs=true/false` : آیا فایل‌های LFS گرفته شوند؟ (پیش‌فرض `true`).  
  `depth=...` : عمق تاریخچه (پیش‌فرض ۱).  
- برای `mirror`:  
  `o=...` : نام پوشه‌ای که سایت در آن ذخیره شود.  
  `depth=...` : عمق خزیدن (پیش‌فرض ۱۰۰۰).  
  **نکته مهم**: آینه‌سازی صفحات HTML، CSS، تصاویر و فایل‌های استاتیک را ذخیره می‌کند. وب‌سایت‌های داینامیک که نیاز به پایگاه داده یا اجرای سمت سرور دارند، ممکن است کاملاً کاربردی نباشند.

## 🧰 روش استفاده

1. **مخزن را Fork کنید** (دکمه Fork بالای صفحه).
2. دو فایل اصلی زیر در فورک شما قرار دارند (نیازی به تغییر نیست، اما می‌توانید سفارشی‌سازی کنید):
   - `downloader.py` در ریشهٔ مخزن
   - `.github/workflows/download.yml` (فایل workflow)
3. به تب **Actions** بروید، روی **Multi Downloader** کلیک کنید.
4. روی دکمهٔ **Run workflow** کلیک کنید.
5. در کادر بزرگ، دستورات خود را خط‌به‌خط وارد کنید (هر خط یک دستور).  
   - می‌توانید از خط‌هایی که با `#` شروع می‌شوند به عنوان کامنت استفاده کنید.
6. (اختیاری) نام فایل ZIP خروجی را در فیلد پایینی تغییر دهید (پیش‌فرض `out.zip`).
7. دکمهٔ سبز **Run workflow** را بزنید.
8. پس از پایان اجرا (علامت ✔ سبز)، به بخش **Artifacts** بروید و فایل ZIP را دانلود کنید.

> **نکته:**  
> در فایل YAML نیز می‌توانید مقادیر پیش‌فرض دستورات را مستقیماً عوض کنید تا هر بار نیاز به تایپ نباشد.

## 📌 نمونه‌های ترکیبی

**دانلود یک فایل + کلون یک مدل از Hugging Face:**
```
wget https://example.com/data.zip o=mydata.zip
git https://huggingface.co/google/vit-base-patch16-224 b=main d=vit lfs=true depth=1
```

**آینه‌سازی چند سایت همزمان:**
```
mirror https://docs.python.org/3/ o=python_docs depth=2000
mirror https://example-site.com o=site2
```

## 🌐 اجرای سایت آینه‌سازی شده (Mirrored Website)

پس از دانلود فایل ZIP و استخراج آن، شما یک کپی محلی از وب‌سایت دارید. بسته به نوع سایت، می‌توانید آن را به صورت لوکال اجرا کنید:

### ۱. سایت‌های استاتیک (HTML/CSS/JS)
ساده‌ترین روش استفاده از **سرور HTTP ساده پایتون** است:
```bash
# ابتدا وارد پوشه آینه‌سازی شده شوید (مثلاً site_mirror)
cd مسیر/به/site_mirror
# اجرای سرور روی پورت 8000
python3 -m http.server 8000
```
حالا در مرورگر خود آدرس `http://localhost:8000` را باز کنید.  
اگر پوشه اصلی سایت داخل یک زیرپوشه (مثلاً `python_docs`) است، باید به آن مسیر بروید.

### ۲. سایت‌های PHP
اگر سایت شامل فایل‌های `.php` بود و می‌خواهید آنها را اجرا کنید، باید PHP روی سیستم شما نصب باشد. سپس از سرور داخلی PHP استفاده کنید:
```bash
# فرض کنید سایت در پوشه my_php_site ذخیره شده
cd مسیر/به/my_php_site
php -S localhost:8080
```
اکنون `http://localhost:8080` را باز کنید.  
توجه: این روش فقط فایل‌های PHP را اجرا می‌کند. اتصال به پایگاه داده یا سرویس‌های جانبی نیاز به راه‌اندازی جداگانه دارد.

### ۳. سایت‌های Node.js / React / Vue و غیره
اگر از آینه‌ساز برای کپی یک سایت استاتیک ساخته‌شده با فریم‌ورک‌های مدرن استفاده کرده‌اید (که خروجی نهایی آنها استاتیک است)، کافی است پوشه `dist` یا `build` را با `python -m http.server` اجرا کنید.  
اما اگر سایت نیاز به سرور Node.js برای APIها دارد، آینه‌ساز فقط فایل‌های استاتیک را ذخیره کرده و بخش داینامیک کار نخواهد کرد.

## 📂 ساختار خروجی ZIP

فایل ZIP نهایی ساختاری مانند زیر دارد:
```
out.zip
├── mydata.zip                (اگر wget باشد)
├── bert/                     (مخزن کلون‌شده)
├── python_docs/              (سایت آینه‌شده)
│   ├── docs.python.org/
│   │   ├── index.html
│   │   ├── ...
├── site2/                    (سایت دوم)
│   ├── ...
```

شما می‌توانید هر بخش را جداگانه استفاده کنید.

## ⚙️ سفارشی‌سازی پیشرفته

- **تغییر دستورات پیش‌فرض**: در فایل `.github/workflows/download.yml` قسمت `default` مربوط به `commands` را ویرایش کنید.
- **تغییر نام خروجی ZIP**: در همان فایل، `out_zip` را تغییر دهید.
- **اضافه کردن وابستگی‌ها**: اگر اسکریپت نیاز به پکیج‌های بیشتری دارد، بخش `pip install` در workflow را به‌روز کنید.

## 🛠️ نکات فنی

- فایل `downloader.py` به هیچ نصب اضافه‌ای جز `requests`، `beautifulsoup4` و `lxml` نیاز ندارد که در workflow نصب می‌شوند.
- تمام عملیات در محیط اوبونتو گیتهاب انجام می‌شود.

- آینه‌ساز به‌طور خودکار لینک‌های داخلی را نسبی می‌کند تا سایت به‌درستی در لوکال اجرا شود.
